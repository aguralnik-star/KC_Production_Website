# First 30 Day Operations Plan

**Project:** K&C Design and Manufacturing Website  
**Launch Date:** 2026-05-22

---

## Overview

This plan defines weekly operational priorities for the first 30 days after production launch. Use the **Go-Live Command Center** (`/admin/go-live`) and **Post-Launch Dashboard** (`/admin/post-launch`) daily.

---

## Week 1 — Verify Systems & Monitor RFQs

**Focus:** Stability and first real RFQ handling

- [ ] Complete Phase 1 and Phase 2 launch checklists
- [ ] Monitor every new RFQ through the 10-step monitoring workflow
- [ ] Review analytics daily (GA4, Clarity)
- [ ] Resolve any launch issues logged as HIGH or CRITICAL
- [ ] Confirm email delivery for all production RFQs
- [ ] Verify admin dashboard availability and RBAC
- [ ] Run daily operational review in post-launch dashboard

---

## Week 2 — Evaluate RFQ Conversion

**Focus:** Conversion funnel and CTA performance

- [ ] Evaluate RFQ submission rate vs. site traffic
- [ ] Review CTA performance on homepage, capabilities, and service pages
- [ ] Identify pages with high traffic but low RFQ starts
- [ ] Improve content or CTAs on underperforming pages if needed
- [ ] Review quote conversion rate and response times
- [ ] Close or resolve MEDIUM severity issues from Week 1

---

## Week 3 — SEO & Content Enrichment

**Focus:** Discoverability and trust signals

- [ ] Review SEO indexing in Google Search Console
- [ ] Confirm sitemap coverage and fix crawl errors
- [ ] Add real manufacturing photos where placeholders remain
- [ ] Expand project examples with approved case studies
- [ ] Review internal linking and service page performance
- [ ] Update meta descriptions if click-through rates are low

---

## Week 4 — Metrics Review & Workflow Optimization

**Focus:** Operational maturity and improvement backlog

- [ ] Review conversion metrics for full 30-day window
- [ ] Review operational workflow efficiency (time to review, time to quote)
- [ ] Prioritize improvements for next development cycle
- [ ] Document lessons learned from first real RFQs
- [ ] Transition from go-live monitoring mode to steady-state operations
- [ ] Archive resolved launch issues and finalize sign-off

---

## Daily Monitoring Checklist (Phase 3)

During all four weeks, monitor:

- New RFQs
- RFQ completion rate
- File upload failures
- Email failures
- Status lookup failures
- Additional info workflow failures
- Public page availability
- Admin dashboard availability
