# Customer Testimonial Approval Guide

**Project:** K&C Design and Manufacturing Website

---

## How to Request Testimonial Approval

1. Identify a satisfied customer willing to provide feedback
2. Draft the testimonial quote in plain language — do not embellish or add unsupported claims
3. Specify exactly where the testimonial will be used (website, proposals, social media, etc.)
4. Send the draft to the customer for review and written approval
5. Log the request in the Real Content admin dashboard (`/admin/real-content`)
6. Do not publish until all checklist items are complete

---

## Approved Wording Process

1. Start with the customer's own words when possible
2. Edit only for clarity, grammar, and length — not to exaggerate
3. Share the final wording with the customer for sign-off
4. Save the approved quote exactly as the customer confirmed
5. Record the approval date and contact in admin notes

---

## Anonymous Testimonial Option

If the customer prefers anonymity:

- Select **anonymous only** in allowed usage
- Use a generic role title (e.g., "Manufacturing Manager") without company name
- Do not include logos or identifying details
- Confirm anonymous display is acceptable in writing

---

## Customer Name / Company Permission

Obtain separate permission for:

- [ ] Use of customer name
- [ ] Use of company name
- [ ] Use of company logo (if applicable)
- [ ] Use on website vs. sales materials vs. social media

Document permission in the admin tracker (`permissionReceived`, `approvalDate`).

---

## Where the Testimonial Will Be Used

Allowed usage options:

- Website
- Sales materials
- Proposals
- Social media
- Anonymous only

Only use the testimonial in channels the customer explicitly approved.

---

## Final Approval Checklist

- [ ] Customer approved quote
- [ ] Customer approved name usage
- [ ] Customer approved company usage
- [ ] No confidential project details
- [ ] No pricing details
- [ ] No unsupported claims
- [ ] Final wording approved
- [ ] Confidentiality reviewed
- [ ] Publish ready flag set
- [ ] Added to `REAL_TESTIMONIALS` in `testimonialsData.js` for publication

---

## If No Approved Testimonial Exists

The public site continues to show **representative testimonials** with a "Representative Example" label. This is the correct behavior until real approved content is available.
